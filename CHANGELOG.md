# Changelog

## 2026-06-17

### 新增

- **需求详情新增「结构化」标签页，点「运行结构化」即对正文与图片做多模态分析，产出带来源的准则、待澄清问题与覆盖账，完成后弹出提示**
- **结构化结果顶部以徽标展示概览：准则总数、确定数、假设数、待澄清数，以及覆盖账的出范围／未覆盖／孤图数量**
- **「待澄清·需先问 PM」问题置顶高亮，每条展示问题、影响、候选答案，以及阻塞的准则与正文出处行号**
- **准则按视图分组展示，每条标注「确定／假设」与验证方式（行为／视觉／数据）、「则」结果、假设说明与溯源行号，关联图片显示缩略图可点开放大**
- **覆盖账分三块展示：非前端／出范围条目（带分类标签）、未覆盖的正文行（带行号）、未挂到任何准则的孤图（缩略图网格）**
- **结构化模型可切换 provider：Claude 或任意 OpenAI 兼容服务（智谱 GLM-4V／通义 Qwen-VL／Gemini／Ollama），未配置模型 key 时运行会提示去配置**

## 2026-06-16

### 新增

- 网页「飞书配置」可填写自建应用 app_id、app_secret 与开放平台域名并保存到本地；配置完整后标题旁显示「就绪」徽标，否则显示「未配置」（app_secret 出于安全不回显，留空表示保持原值）
- 网页粘贴飞书文档链接（/docx/ 或 /wiki/）后可解析正文与图片并落盘，完成后弹出提示并显示解析到的图片张数
- 未填好飞书凭据时，解析链接的输入框与按钮禁用，并提示先去「飞书配置」填写
- 网页左侧「已解析需求」列表展示每条记录的标题、blocks 数与图片数，点击可切换右侧查看的需求
- 网页右侧以「正文」「图片」两个标签展示解析结果：正文可滚动查看，图片以网格展示并可点开放大，标题旁可一键跳转飞书原文
